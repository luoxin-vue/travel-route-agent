# 项目简报：ROUTE_SYSTEM PWA

## 项目概述
**ROUTE_SYSTEM** 是一款高保真的旅游路线规划 PWA（渐进式 Web 应用），采用 AI 原生、对话优先的交互界面。它将开发工具的实用主义美学与现代排版设计的温度相结合，灵感源自 "Claude Code" 的设计风格。

## 目标受众
- 偏好结构化、高效规划的科技爱好者。
- 欣赏极简主义、信息密集型界面的用户。
- 寻求与 AI 代理协作合成行程的旅行者。

## 核心视觉识别
- **风格原型**：“终端社论风 (Terminal Editorial)” —— 结合了 CLI（命令行界面）的精准感与高端排版设计。
- **配色方案**：
  - **主表面**：`#FCF9F8`（温暖的乳白色/米色）
  - **品牌色**：`#D97757`（铁锈红/陶土色）
  - **文字**：深板岩色和单色灰色，用于构建层级。
- **字体**：正文使用 Inter (Sans-serif)；系统状态、坐标和协议 ID 使用等宽字体 (Monospace)。
- **UI 元素**：低圆角半径 (`ROUND_FOUR`)、细微边框以及 `[Command]` 风格的按钮。

## 功能需求
### 1. 对话式代理界面 (会话)
- **AI 代理 (CONCIERGE_DAEMON)**：一个结构化的聊天界面，代理在此提议行程。
- **系统遥测**：显示系统状态（如 CPU_空闲、延迟），强化技术主题感。
- **命令输入**：终端风格的提示符 (`>`) 用于用户输入。

### 2. 结构化行程管理 (计划)
- **垂直时间轴**：旅游节点（交通、住宿、活动）的逻辑流。
- **元数据密度**：每个节点包含精准数据：坐标、时间戳、预约 ID 和协议类型。
- **可操作节点**：受 CLI 启发的按钮，如 `[编辑]`、`[标记]` 和 `[入住]`。

### 3. 环境天气与物流
- 实时数据集成，如天气预报和交通衔接跟踪（例如：火车平台分配）。

## 设计系统参考
- **设计系统**：`{{DATA:DESIGN_SYSTEM:DESIGN_SYSTEM_1}}` (Anthesis Travel Logic)
- **关键组件**：
  - `TopAppBar`：带有系统图标的极简设计。
  - `BottomNavBar`：清晰的导航标签（会话、计划、地图、账户）。
  - `RouteTimeline`：行程可视化的核心结构元素。

## 本地化
- 全面支持中文（简体）和英文界面，在不同语言下保持统一的技术“协议”感。
