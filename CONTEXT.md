# Travel Route Agent (ROUTE_SYSTEM)

AI 原生、精炼克制的旅游路线规划 Agent 系统。遵循 Warm Linen & Terracotta（暖感生活空气感）设计风格与 Ponytail 极简代码工程规范。

## 领域词汇规范 (Language & Tone)

**Agent角色 (Concierge Daemon)**:
精确、克制、纯粹输出事实与结构化行程数据的随心旅行管家内核。
_Avoid_: AI助手, 智能客服, 人工智能机器人, 黑客终端

**交互对话 (Dialogue)**:
冷静、直接、以数据和点位为主的要点汇报，严禁任何套话、客套与自述。
_Avoid_: 祝您旅途愉快, 希望能帮到您, 很高兴为您服务, 作为AI

**路线行程 (Itinerary)**:
由按天和时间节点构成的结构化 JSON 实体及对应的可视化地图点位。国际行程节点支持自动配图。
_Avoid_: 建议方案, 方案草案, 推荐列表

**图片资源 (Image Resources)**:
- **图片渠道 / Image Source**: 提供图片搜索与获取的外部服务（Unsplash、Pexels、Flickr、高德地图等）。每个渠道有独立适配器。
- **代理图片 / Proxied Image**: 经后端 `/api/img` 代理获取、缓存 24h 的图片。当前限高德、Wikipedia、Wikimedia、Pexels。
- **引用图片 / Referenced Image**: 第三方外链 URL，前端直接 `<img>` 展示，不经过代理不缓存。用于 Unsplash、Flickr 等渠道的结果。
- **渠道适配器 / Source Adapter**: 封装单个图片渠道搜索接口的模块，统一输入（关键词）和输出（图片 URL 列表）。
- **自动配图 / Auto Image Matching**: Agent 生成国际行程时自动从已启用渠道中搜索节点图片，取 Top1 结果绑定。
- **通用占位图 / Default Placeholder**: 当所有渠道搜索均无结果时，节点使用的降级展示图。

## 决策记录 (Decisions)

- **去AI味范围**: 全面覆盖 Prompt 语气（去AI化客套）、后端/前端代码注释（清理AI废话注释与过度封装）、UI 视觉（审核去除AI样板元素与黑客终端感）。
- **Agent语气标准**: 选项 A。开门见山直接输出数据与点位，强制黑名单禁用所有AI客服套话（“作为AI”、“希望有所帮助”、“祝您旅途愉快”等），禁止前言和结语。
- **代码与注释标准**: 方案 A。彻底删除字面同义反复的AI注释，仅保留关键业务机制中文注释；清理过度解耦的纯套壳UI小组件，降低抽象层数。
- **行程节点编辑与新增**: 采用 `NodeEditModal` 模态弹窗形式，支持对节点的完整字段编辑（名称、类型、时间、协议、备注、图片等）、带二次确认的节点删除，以及复用同一弹窗进行“添加新游玩节点”。
- **全局偏好与设置**: 采用 `SettingsModal` 综合弹窗，划分“旅行偏好”、“数据管理”、“系统与地图”三大板块，偏好信息自动持久化。
- **坐标信息界面替换**: 隐去卡片上无实际用处的数字化经纬度坐标（`39.9042° N, 116.4074° E`），在 `StopCard` 及编辑弹窗中替换为更贴合旅游场景的“区域位置描述 (`location_summary`)”与“建议游玩/停留时长 (`duration`)”，底层 `lat`/`lng` 仅供高德地图画点渲染使用。
- **相邻站距离与出行时间提示**: 废弃无结构字符串 `prev_distance`，改为结构化数值 `next_distance_km`（本站到下一站的距离）。所有非末站统一展示“距下站”，末站 `next_distance_km` 为 null 时不展示距离提示，自然过渡到区域位置 (`location_summary`) 与建议游玩时长 (`duration`)。出行时间基于路线距离及出行方式（`protocol`）配合速度表动态计算；当用户手动选择的出行方式与系统推荐不一致时，同时呈现建议与当前方式的耗时对比。
- **国际行程多渠道自动配图**: 国际目的地（高德无 POI 数据）通过后端聚合搜索 Unsplash、Pexels、Flickr (CC) 获取节点配图。结果以引用图片（外链直连、不代理不缓存）返回，Agent 生成行程时自动取 Top1 绑定。所有渠道零结果时降级为通用占位图。高德/Wikipedia 等现有代理图片链路不变。





## 地图显示术语

- **默认底图**：无论界面使用浅色或深色主题，地图始终使用高德默认样式，确保城市、街道和道路信息可见；行程坐标点与路线连线作为覆盖物继续显示。
